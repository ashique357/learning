# Install and Set Up kubectl

Think of kubectl as your **remote control** for Kubernetes - just like how you use a TV remote to control your television from your couch, kubectl lets you control your Kubernetes cluster from your computer.

## What is kubectl?

kubectl (pronounced "kube-control" or "kube-c-t-l") is like having a **universal remote** that can:
- Turn services on/off (start/stop applications)
- Check what's running (see status of your apps)
- Change settings (modify configurations)
- Get information (view logs and details)

## Installation Methods

### 1. Install kubectl Binary
Like downloading an app on your phone:

```bash
# Linux
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl

# macOS
brew install kubectl

# Windows
choco install kubernetes-cli
```

### 2. Install with Package Managers
Like using an app store:

```bash
# Ubuntu/Debian
sudo apt-get update && sudo apt-get install -y kubectl

# CentOS/RHEL
sudo yum install -y kubectl

# Snap
sudo snap install kubectl --classic
```

## Setting Up Your First Cluster

### Option 1: Minikube (Local Development)
Think of Minikube as a **practice driving course** - it creates a small Kubernetes cluster on your laptop for learning.

```bash
# Install Minikube
curl -Lo minikube https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube /usr/local/bin/

# Start your local cluster
minikube start

# Check if it's working
kubectl get nodes
```

### Option 2: Kind (Kubernetes in Docker)
Kind is like having a **simulator** - it runs Kubernetes inside Docker containers.

```bash
# Install Kind
go install sigs.k8s.io/kind@v0.20.0

# Create a cluster
kind create cluster --name my-cluster

# Use the cluster
kubectl cluster-info --context kind-my-cluster
```

### Option 3: Cloud Providers
Like renting a **professional race track** instead of practicing in your driveway:

```bash
# AWS EKS
aws eks update-kubeconfig --region us-west-2 --name my-cluster

# Google GKE
gcloud container clusters get-credentials my-cluster --zone us-central1-a

# Azure AKS
az aks get-credentials --resource-group myResourceGroup --name myAKSCluster
```

## Configuring kubectl

### Understanding kubeconfig
kubeconfig is like your **address book** - it stores information about:
- **Clusters**: Where your Kubernetes clusters are located
- **Users**: Your login credentials
- **Contexts**: Which cluster + user combination to use

```yaml
# ~/.kube/config example
apiVersion: v1
kind: Config
clusters:
- cluster:
    server: https://my-cluster.example.com
  name: my-cluster
users:
- name: my-user
  user:
    token: abc123...
contexts:
- context:
    cluster: my-cluster
    user: my-user
  name: my-context
current-context: my-context
```

### Managing Multiple Clusters
Like having multiple TV remotes for different rooms:

```bash
# See all available contexts (clusters)
kubectl config get-contexts

# Switch between clusters
kubectl config use-context production-cluster
kubectl config use-context development-cluster

# Check current context
kubectl config current-context

# Set default namespace
kubectl config set-context --current --namespace=my-app
```

## Verifying Your Setup

### Basic Health Checks
Like checking if your remote control batteries are working:

```bash
# Check kubectl version
kubectl version --client

# Check cluster connection
kubectl cluster-info

# List all nodes
kubectl get nodes

# Check cluster health
kubectl get componentstatuses
```

### Testing Basic Operations
Like testing all buttons on your remote:

```bash
# Create a test pod
kubectl run test-pod --image=nginx --restart=Never

# Check if it's running
kubectl get pods

# Get detailed information
kubectl describe pod test-pod

# Clean up
kubectl delete pod test-pod
```

## Common Setup Issues

### 1. Connection Problems
**Problem**: "Unable to connect to the server"
**Solution**: Like checking if your WiFi is connected
```bash
# Check if cluster is running
kubectl cluster-info

# Verify kubeconfig
kubectl config view

# Check network connectivity
ping your-cluster-endpoint
```

### 2. Permission Issues
**Problem**: "Forbidden" or "Unauthorized"
**Solution**: Like making sure you have the right key card
```bash
# Check current user
kubectl auth whoami

# Test permissions
kubectl auth can-i get pods

# Check role bindings
kubectl get rolebindings
```

### 3. Context Confusion
**Problem**: Commands affecting wrong cluster
**Solution**: Like making sure you're using the right remote for the right TV
```bash
# Always check current context
kubectl config current-context

# Use explicit context
kubectl --context=dev-cluster get pods
```

## Advanced Configuration

### Custom kubectl Configuration
```bash
# Set custom kubeconfig location
export KUBECONFIG=/path/to/my/config

# Merge multiple configs
export KUBECONFIG=~/.kube/config:~/.kube/config-cluster2

# Create new context
kubectl config set-context dev --cluster=development --user=dev-user --namespace=development
```

### Useful Aliases and Shortcuts
```bash
# Add to ~/.bashrc or ~/.zshrc
alias k=kubectl
alias kgp='kubectl get pods'
alias kgs='kubectl get services'
alias kgd='kubectl get deployments'

# Enable kubectl autocompletion
source <(kubectl completion bash)
```

## Security Best Practices

### 1. Protect Your kubeconfig
- Store in secure location (`~/.kube/config`)
- Set proper file permissions: `chmod 600 ~/.kube/config`
- Never commit kubeconfig to version control
- Use separate configs for different environments

### 2. Use Service Accounts
```bash
# Create service account instead of using admin credentials
kubectl create serviceaccount my-app-sa

# Bind specific permissions
kubectl create rolebinding my-app-binding --clusterrole=view --serviceaccount=default:my-app-sa
```

### 3. Regular Credential Rotation
```bash
# Check token expiration
kubectl config view --raw -o jsonpath='{.users[0].user.auth-provider.config.expiry}'

# Refresh credentials (cloud providers)
aws eks update-kubeconfig --region us-west-2 --name my-cluster
```
