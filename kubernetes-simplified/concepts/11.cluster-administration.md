# Kubernetes Cluster Administration 🏛️

## What is Cluster Administration? (Simple Explanation)

Think of cluster administration like **managing a smart city**!

### Real-Life Example 🏙️
- **City Manager (Cluster Admin)** - Oversees entire city operations
- **Infrastructure** - Roads, power, water (nodes, networking, storage)
- **City Services** - Police, fire, utilities (system pods, monitoring)
- **Maintenance** - Regular upkeep and upgrades
- **Emergency Response** - Handle disasters and outages

### Advanced Cluster Management

#### Multi-Cluster Management
```yaml
# cluster-api-cluster.yaml
apiVersion: cluster.x-k8s.io/v1beta1
kind: Cluster
metadata:
  name: pizza-production
  namespace: default
spec:
  clusterNetwork:
    pods:
      cidrBlocks: ["192.168.0.0/16"]
  infrastructureRef:
    apiVersion: infrastructure.cluster.x-k8s.io/v1beta1
    kind: AWSCluster
    name: pizza-production
  controlPlaneRef:
    kind: KubeadmControlPlane
    apiVersion: controlplane.cluster.x-k8s.io/v1beta1
    name: pizza-production-control-plane
```

#### GitOps Cluster Management
```yaml
# argocd-application.yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: pizza-infrastructure
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/pizza-company/k8s-infrastructure
    targetRevision: HEAD
    path: clusters/production
  destination:
    server: https://kubernetes.default.svc
    namespace: kube-system
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
```

## 3. Backup and Recovery 💾

### etcd Backup (Critical!)
```bash
# Backup etcd (the brain of Kubernetes)
ETCDCTL_API=3 etcdctl snapshot save backup.db \
  --endpoints=https://127.0.0.1:2379 \
  --cacert=/etc/kubernetes/pki/etcd/ca.crt \
  --cert=/etc/kubernetes/pki/etcd/healthcheck-client.crt \
  --key=/etc/kubernetes/pki/etcd/healthcheck-client.key

# Verify backup
ETCDCTL_API=3 etcdctl --write-out=table snapshot status backup.db

# Restore from backup (emergency only!)
ETCDCTL_API=3 etcdctl snapshot restore backup.db \
  --data-dir /var/lib/etcd-from-backup
```

### Application Backup Example
```yaml
# backup-job.yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: database-backup
spec:
  schedule: "0 2 * * *"  # Daily at 2 AM
  jobTemplate:
    spec:
      template:
        spec:
          containers:
          - name: backup
            image: postgres:13
            command:
            - /bin/bash
            - -c
            - |
              pg_dump -h database-service -U postgres myapp > /backup/backup-$(date +%Y%m%d).sql
              # Upload to cloud storage
              aws s3 cp /backup/backup-$(date +%Y%m%d).sql s3://my-backups/
            env:
            - name: PGPASSWORD
              valueFrom:
                secretKeyRef:
                  name: db-secret
                  key: password
            volumeMounts:
            - name: backup-storage
              mountPath: /backup
          volumes:
          - name: backup-storage
            emptyDir: {}
          restartPolicy: OnFailure
```

## 4. Cluster Upgrades 🔄

### Upgrade Process (Managed Clusters)
```bash
# Check current version
kubectl version

# For EKS (AWS)
aws eks update-cluster-version --region us-west-2 --name my-cluster --kubernetes-version 1.28

# For GKE (Google Cloud)
gcloud container clusters upgrade my-cluster --master --cluster-version 1.28.0

# For AKS (Azure)
az aks upgrade --resource-group myResourceGroup --name myAKSCluster --kubernetes-version 1.28.0
```

### Manual Upgrade Steps
```bash
# 1. Upgrade control plane first
kubeadm upgrade plan
kubeadm upgrade apply v1.28.0

# 2. Upgrade kubelet on each node
kubectl drain node1 --ignore-daemonsets
# SSH to node1
apt-get update && apt-get install -y kubelet=1.28.0-00 kubectl=1.28.0-00
systemctl daemon-reload && systemctl restart kubelet
# Back to master
kubectl uncordon node1

# 3. Repeat for all nodes
```

## 5. Logging and Monitoring 📋

### Centralized Logging Setup
```yaml
# logging-stack.yaml
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: fluentd
  namespace: kube-system
spec:
  selector:
    matchLabels:
      name: fluentd
  template:
    metadata:
      labels:
        name: fluentd
    spec:
      containers:
      - name: fluentd
        image: fluent/fluentd-kubernetes-daemonset:v1-debian-elasticsearch
        env:
        - name: FLUENT_ELASTICSEARCH_HOST
          value: "elasticsearch.logging.svc.cluster.local"
        - name: FLUENT_ELASTICSEARCH_PORT
          value: "9200"
        volumeMounts:
        - name: varlog
          mountPath: /var/log
        - name: varlibdockercontainers
          mountPath: /var/lib/docker/containers
          readOnly: true
      volumes:
      - name: varlog
        hostPath:
          path: /var/log
      - name: varlibdockercontainers
        hostPath:
          path: /var/lib/docker/containers
```

### Monitoring with Prometheus

## 7. Disaster Recovery 🚨

### Disaster Recovery Plan
```yaml
# disaster-recovery-checklist.yaml
# 1. Regular Backups
apiVersion: batch/v1
kind: CronJob
metadata:
  name: cluster-backup
spec:
  schedule: "0 1 * * *"  # Daily at 1 AM
  jobTemplate:
    spec:
      template:
        spec:
          containers:
          - name: backup
            image: backup-tool:latest
            command:
            - /bin/bash
            - -c
            - |
              # Backup etcd
              etcdctl snapshot save /backup/etcd-$(date +%Y%m%d).db
              
              # Backup all resources
              kubectl get all --all-namespaces -o yaml > /backup/resources-$(date +%Y%m%d).yaml
              
              # Upload to remote storage
              aws s3 sync /backup/ s3://disaster-recovery-bucket/
          restartPolicy: OnFailure
```

### Recovery Procedures
```bash
# 1. Restore etcd from backup
systemctl stop etcd
rm -rf /var/lib/etcd
etcdctl snapshot restore backup.db --data-dir /var/lib/etcd
systemctl start etcd

# 2. Restore applications
kubectl apply -f resources-backup.yaml

# 3. Verify cluster health
kubectl get nodes
kubectl get pods -A
```

## 8. Performance Tuning ⚡

### Cluster Optimization
```yaml
# performance-tuning.yaml
# Optimize scheduler
apiVersion: kubescheduler.config.k8s.io/v1beta3
kind: KubeSchedulerConfiguration
profiles:
- schedulerName: default-scheduler
  plugins:
    score:
      enabled:
      - name: NodeResourcesFit
      - name: NodeAffinity
  pluginConfig:
  - name: NodeResourcesFit
    args:
      scoringStrategy:
        type: LeastAllocated
```

### Resource Optimization
```bash
# Check resource waste
kubectl top nodes
kubectl describe nodes | grep -A 5 "Allocated resources"

# Optimize pod resources
kubectl get pods -A -o custom-columns=NAME:.metadata.name,CPU_REQ:.spec.containers[*].resources.requests.cpu,MEM_REQ:.spec.containers[*].resources.requests.memory
```

## 9. Troubleshooting Guide 🔧

### Common Issues and Solutions

#### Cluster Issues
```bash
# API server not responding
kubectl cluster-info
systemctl status kubelet
journalctl -u kubelet

# Node not ready
kubectl describe node <node-name>
systemctl status kubelet
systemctl status docker
```

#### Network Issues
```bash
# Pod networking problems
kubectl get pods -o wide
kubectl describe pod <pod-name>
kubectl logs <pod-name>

# DNS issues
kubectl run test-pod --image=busybox --rm -it -- nslookup kubernetes.default
```

#### Storage Issues
```bash
# PVC stuck in pending
kubectl describe pvc <pvc-name>
kubectl get storageclass
```

## 10. Complete Admin Example 🎯

### Production Cluster Setup
```yaml
# production-cluster-config.yaml
# Monitoring namespace
apiVersion: v1
kind: Namespace
metadata:
  name: monitoring
---
# Logging namespace
apiVersion: v1
kind: Namespace
metadata:
  name: logging
---
# Backup namespace
apiVersion: v1
kind: Namespace
metadata:
  name: backup
---
# Resource quotas for system namespaces
apiVersion: v1
kind: ResourceQuota
metadata:
  name: monitoring-quota
  namespace: monitoring
spec:
  hard:
    requests.cpu: "4"
    requests.memory: 8Gi
    limits.cpu: "8"
    limits.memory: 16Gi
---
# Cluster monitoring deployment
apiVersion: apps/v1
kind: Deployment
metadata:
  name: cluster-monitor
  namespace: monitoring
spec:
  replicas: 1
  selector:
    matchLabels:
      app: cluster-monitor
  template:
    metadata:
      labels:
        app: cluster-monitor
    spec:
      containers:
      - name: monitor
        image: prom/prometheus:latest
        ports:
        - containerPort: 9090
        resources:
          requests:
            memory: "2Gi"
            cpu: "1"
          limits:
            memory: "4Gi"
            cpu: "2"
```

### Deployment Script
```bash
#!/bin/bash
# deploy-production-cluster.sh

echo "🏛️ Setting up production cluster administration..."

# 1. Create system namespaces
kubectl apply -f production-cluster-config.yaml

# 2. Set up monitoring
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm install prometheus prometheus-community/kube-prometheus-stack -n monitoring

# 3. Set up logging
helm repo add elastic https://helm.elastic.co
helm install elasticsearch elastic/elasticsearch -n logging
helm install kibana elastic/kibana -n logging

# 4. Set up backup jobs
kubectl apply -f backup-jobs.yaml

# 5. Configure alerts
kubectl apply -f alert-rules.yaml

echo "✅ Production cluster administration setup complete!"
```

- Monitor cluster health and resource usage

- Review logs for errors and warnings
- Verify backup completion

- Review resource quotas and usage trends
- Update security policies and RBAC
- Check certificate expiration dates
- Test disaster recovery procedures

- Plan and execute cluster upgrades
- Review and optimize resource allocation
- Audit security configurations
